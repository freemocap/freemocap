v1.8.0 adds an optional outlier-rejection step to the triangulation pipeline, which should greatly improve the reconstruction quality with 4+ camera systems!
Note that it is set OFF by default in this release — We are pretty sure it will improve results in most cases, but because it touches the core processing pipeline we wanted to be careful about rolling it out.
If you have a 4+ camera set up (or even better, old recordings that didn't quite reconstruct like they should), please try processing with the new setting turned and let us know how it goes on this tracking issue on Github: freemocap#782 or in the FreeMoCap Discord server.

What it does - Rejects high-error camera-keypoint views during triangulation

When triangulating a 3D point, the method checks how well the reconstructed point projects back into each camera view. Points with high reprojection error are re-triangulated using a subset of cameras — an exponential weighting scheme scores candidate subsets by their reprojection error and selects the combination that minimizes error across the remaining views.
More details on the Reprojection Filtering docs page.
See PR #758 for the code.

Why it matters - Allows for higher camera count (4+) recordings

Previously, adding more cameras to a rig ran into a "poison pill" problem: the more cameras you had, the higher the odds that at least one of them would pick up a bad track (spooky ghost skeletons💀👻) and contaminate the 3D reconstruction. This effectively capped the number of cameras that were useful in practice. With outlier rejection, bad views are automatically identified and excluded on a per-point basis, making higher camera count reconstruction more robust.

How to use it - Check the box at `Process Data > 3d Triangulation > User Outlier Rejection Method?` before processing data

We recommend using this method for any recording over 4 cameras. For 3 camera recordings, this method may result in unstable reconstructions (because 2-camera triangulation can be shakey) For the method to have any effect, the number of cameras in your recording must be greater than "Minimum Cameras for Triangulation" (see below) — otherwise there are no subsets left to test.

🆕 Also new: Minimum Cameras for Triangulation (a top-level option)

While we were in the neighborhood, we also added a "Minimum Cameras for Triangulation" option to class="emphasis">3D Triangulation Methods panel with a default value of 3. It now applies to both the simple triangulation path and the outlier-rejection path. Any point with fewer valid 2D detections than this threshold will be left out of the 3D output.

If you set this value higher than the number of cameras in your recording (e.g. min=3 on a 2-camera rig), FreeMoCap will emit a warning in the logs and automatically adjust the value down to the number of cameras available so processing still runs.
